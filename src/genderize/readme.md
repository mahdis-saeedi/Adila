## Generating `females.csv`
This repository provides guidelines for generating a `females.csv` file, which contains the column indices of female experts for a given filtered dataset.

---

For the full dataset, gender information is provided through the `c2g.pkl` file, which maps each candidate to a gender label. 



When a dataset is filtered, the ordering and column indices of experts may change. As a result, the original gender mapping no longer aligns with the filtered dataset.

The `females.csv` file for a filtered dataset can be generated by running the `generate_i2g_and_female_csv` function in `i2g.py`, using the following inputs:

![#3282F6](https://placehold.co/15x15/3282F6/3282F6.png) **Inputs:**

- `c2g.pkl`: Candidate-to-gender mapping for the *full dataset*, available on Hugging Face for: [`dblp`](https://huggingface.co/datasets/fani-lab/OpeNTF/tree/main/output/dblp/dblp.v12.json), [`uspt`](https://huggingface.co/datasets/fani-lab/OpeNTF/tree/main/output/uspt/patent.tsv), and [`imdb`](https://huggingface.co/datasets/fani-lab/OpeNTF/tree/main/output/imdb/title.basics.tsv).

- `c2i.pkl`: Candidate-to-index mapping for the *filtered dataset*, generated in the corresponding filtering output directory (e.g., `/OpeNTF/output/dblp/dblp.v12.json.mt10.ts2`).

![#3282F6](https://placehold.co/15x15/3282F6/3282F6.png) **Output:**

  `female.csv:` Contains the column indices of female experts in the filtered dataset.

---

**Notes:**

- A separate `females.csv` file must be generated for each filtered dataset.
- Ensure that `c2g.pkl` corresponds to the same base dataset as the filtered dataset.
